# Project Memory

## Principios

- SDD primero: cada cambio relevante nace de una spec.
- Prompts versionados: todo prompt operativo vive en `prompts/`.
- Skills pequeñas: una skill debe resolver un tipo de tarea claro.
- Harness siempre: cambios de comportamiento deben traer evaluación.
- Autonomía auditada: el agente actúa con trazabilidad.

## Integraciones objetivo

- Harness.io para implementación y soporte autónomo.
- GitHub Actions para CI de PR.
- Jira para trabajo.
- Confluence para documentación.
- Figma para diseño.
- Slack para notificaciones.
- Azure/AWS para despliegue.
