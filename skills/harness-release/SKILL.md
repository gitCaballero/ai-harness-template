# Harness Release

## Cuándo usar

Usa esta skill para preparar una liberación con Harness.io o GitHub Actions.

## Procedimiento

1. Confirmar cumplimiento de `CONSTITUTION.md`.
2. Confirmar uso consistente de `ai-engineering/GLOSSARY.md`.
3. Ejecutar `npm run ai:harness`.
4. Revisar guardrails.
5. Verificar estado de implementación y memoria actualizada.
6. Verificar checklist end-to-end.
7. Preparar evidencia de release.
8. Confirmar aprobaciones por ambiente.
9. Promover con el pipeline configurado.
